# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please report it responsibly.

### How to Report

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to **security@dacolabs.com** with the following information:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: We will acknowledge receipt of your report within 48 hours
- **Initial Assessment**: We will provide an initial assessment within 5 business days
- **Resolution Timeline**: We aim to resolve critical vulnerabilities within 30 days
- **Communication**: We will keep you informed of our progress

### Disclosure Policy

- We request that you give us reasonable time to address the vulnerability before public disclosure
- We will coordinate with you on the disclosure timeline
- We will credit you for the discovery (unless you prefer to remain anonymous)

## Security Best Practices

When using Daco CLI:

- Keep the CLI updated to the latest stable version
- Review data product definitions before applying them
- Use environment variables or secure vaults for sensitive configuration
- Follow the principle of least privilege when configuring access
